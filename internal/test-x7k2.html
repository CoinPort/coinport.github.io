How to use this document

Work through each section in order. For every step, mark PASS or FAIL and add a note for any unexpected behaviour, console errors, or visual glitches. Capture a screenshot on FAIL.

A flow is PASS only if every step inside it passes. If a flow fails, log a bug ticket with: section number, step number, screenshot, browser console output (red errors), and the failing API response from DevTools Network tab.

Test data preparation (do this once before starting):

  • Two test accounts: Account A — fully KYC-verified (level 3), holds AUD + at least one crypto, has at least one approved AUD beneficiary. Account B — fresh signup, no verification, no beneficiaries.
  • A bank account you can receive AUD into (BSB + account number, OR a PayID).
  • An external crypto wallet address for at least BTC and USDT.
  • Authenticator app (Google Authenticator / Authy) ready.

General notes:

  • Open DevTools → Console tab. If a red error appears at any step, mark FAIL and attach the message.
  • Open DevTools → Network tab → Fetch/XHR filter. If any request returns 4xx or 5xx during a flow, mark FAIL and attach the response body.
  • For each modal that opens, verify the Close (X) button works AND clicking outside the modal works.

State is saved to your browser's localStorage automatically — closing the tab won't lose progress. Keep using the same browser to keep your results.

When you're done, click Email results to QA at the top of the page. Your default mail client will open pre-filled with a structured summary (failed steps, notes, full JSON) addressed to testing@coinport.com.au — just hit Send. Export results (JSON) and Print / save as PDF are alternative ways to capture your results.
